# Random Password Generator
**Автор:** Летова Наталья
Графическое приложение на Python для генерации надёжных случайных паролей. Позволяет настраивать длину и состав пароля, сохранять историю генераций в формате JSON и копировать результат в буфер обмена.

## Функциональность
- **Генерация паролей:** Создание случайного пароля с выбором длины (от 4 до 64 символов) и типов символов:
  - Цифры (0-9)
  - Буквы (A-Z, a-z)
  - Спецсимволы (!, @, #, $, ...)
- **Гарантия состава:** Приложение гарантирует, что в сгенерированном пароле будет хотя бы один символ из каждой выбранной категории.
- **Валидация ввода:** Полная проверка введённых данных:
  - Длина пароля должна быть числом в заданном диапазоне.
  - Должен быть выбран хотя бы один тип символов.
- **Сохранение истории:** Автоматическое сохранение каждого сгенерированного пароля в файл `history.json`.
- **Просмотр истории:** Удобная таблица со временем генерации, паролем, длиной и опциями.
- **Очистка истории:** Возможность удалить все записи из файла истории.
- **Копирование в буфер:** Кнопка для быстрого копирования сгенерированного пароля в буфер обмена.


## Инструкция по запуску
### Требования
- Python 3.6 или новее.
- Стандартная библиотека `tkinter` (обычно входит в состав Python при установке).

Примеры использования
Обычная генерация: Установите длину "14", отметьте "Цифры", "Буквы" и "Спецсимволы". Нажмите "Сгенерировать пароль".

Генерация цифрового PIN-кода: Установите длину "6", оставьте отмеченным только чекбокс "Цифры". Нажмите "Сгенерировать пароль".

Просмотр истории: Прокрутите таблицу внизу окна, чтобы увидеть ранее сгенерированные пароли. Все данные загружаются из файла history.json при старте программы.

Копирование пароля: После генерации нажмите кнопку "Копировать", и пароль окажется в буфере обмена.

Тестирование
Программа была протестирована на следующих сценариях:

Позитивные: Генерация паролей со всеми комбинациями опций. Сохранение и загрузка истории из корректного JSON-файла.

Негативные: Попытка ввести длину 0, -5, 100, текст. Генерация без выбора ни одного типа символов. Попытка загрузить данные из повреждённого JSON-файла. Всё обрабатывается с выводом понятных сообщений об ошибках через графический интерфейс.

Граничные: Генерация паролей минимально (4) и максимально (64) допустимой длины.
